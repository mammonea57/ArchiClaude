Optimisation du footprint — identification des segments

Comment identifier voirie / séparatives / fond de parcelle pour appliquer les reculs différenciés ?

A

Détection par données externes (BAN + BDTopo routes)

On interroge la BD TOPO pour trouver les routes à proximité. Les segments de la parcelle qui bordent une route = voirie. Les autres segments adjacents à d'autres parcelles = séparatives latérales. Le segment le plus éloigné de la voirie principale = fond.

Avantages

  • Précis — utilise les données réelles du réseau routier
  • Gère les parcelles d'angle (2 côtés voirie)
  • Distingue voie publique vs chemin privé

Inconvénients

  • Requête BDTopo supplémentaire (déjà disponible Phase 1)
  • Routes pas toujours parfaitement alignées avec les limites cadastrales
B

Détection géométrique pure (heuristique)

Sans données externes : le segment le plus long face à un espace ouvert = voirie (hypothèse). Les segments courts adjacents = séparatives. Le segment opposé au plus long = fond.

Avantages

  • Aucune requête externe nécessaire
  • Rapide, pas de dépendance réseau

Inconvénients

  • Faux positifs sur parcelles irrégulières
  • Ne distingue pas voie publique vs chemin privé
  • Échoue sur parcelles en drapeau ou enclavées
C

Hybride : BDTopo routes + heuristique fallback

On utilise les routes BDTopo quand disponibles (option A). Si pas de route trouvée à proximité (parcelle enclavée, données manquantes), on bascule sur l'heuristique géométrique (option B) avec un warning "classification segments approximative".

Avantages

  • Meilleure précision possible (données réelles quand dispo)
  • Jamais bloqué (fallback heuristique)
  • Warning explicite quand approximation

Inconvénients

  • Plus de code à maintenir (2 stratégies)
